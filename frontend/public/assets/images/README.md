# 🛡️ CypherVault

> **The CypherVault Initiative**
> A dual-state architecture: half impenetrable fortress, half intentional sandbox. Engineered for operators to master the art of digital defense by studying the fractures in vulnerable routing protocols.

## 🚀 Core Architecture
* **Mainframe:** Node.js
* **API Core:** Express.js
* **Vault (Database):** Supabase (PostgreSQL)
* **Client:** Vanilla JS with Glassmorphism & Terminal Aesthetics
* **AI Core:** S.W.A.L.E. Virtual Intelligence Proxy

## 🎯 Supported Exploit Modules (Advanced Cyber Range)
CypherVault contains intentionally vulnerable endpoints paired with cryptographically secured equivalents to demonstrate proper mitigation:
* **Authentication Bypass (SQLi):** Exploit vulnerable database queries.
* **Information Disclosure & Client-Side Auth Bypass:** Local storage and source code manipulation.
* **Cross-Site Request Forgery (CSRF):** Demonstrates cryptographic token armor.
* **Local File Inclusion (LFI):** Path traversal exploits.
* **Server-Side Request Forgery (SSRF):** Internal network pivoting.
* **Cross-Site Scripting (XSS):** Payload injection.
* **Command Injection (RCE):** Remote Code Execution diagnostics *(Pending/Incoming)*.

## ⚙️ Initialization Sequence

```bash
# 1. Clone the repository
git clone [https://github.com/Swarup-Ingale/CypherVault.git]

# 2. Navigate to the core directory
cd CypherVault/backend

# 3. Install dependencies
npm install

# 4. Configure Environment
# Create a .env file with your Supabase credentials, GEMINI_API_KEY, and Session Secrets.

# 5. Boot the Mainframe
npm run dev

# ⚠️ Directive / Rules of Engagement
CypherVault is a strictly safe, controlled educational sandbox environment. Do not deploy the vulnerable endpoints in a live production environment without strict network isolation. All testing must be conducted on systems you own or have explicit permission to audit.
